Technical Background

I call myself a Software Engineer. I like every part of Software Engineering: frontend, backend, devops.

While I mostly have a Java/Oracle SQL backend, I like also doing frontend and DevOps. I like to have my projects and code crisp and clear, I value code clarity, cohesion, decoupling.

I'm strongly opinionated on how things should be done, but I also know that our job is made of tradeoff and I'm not a zealot by any means.

In every project I try to stick with my three pillars of good Software Engineering:

  1. Cohesion and Decoupling
  2. Good Naming
  3. Code Readability
